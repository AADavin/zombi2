# Basic diversification (birth–death)

The species tree is generated by a **birth–death process**: lineages speciate at rate `birth` (λ)
and go extinct at rate `death` (μ). ZOMBI2 draws it either **backward** — a reconstructed tree
conditioned on the number of extant tips, with internal-node ages from the reconstructed-process CDF
(Hartmann, Wong & Stadler 2010) — or **forward** in time, keeping extinct lineages. This page covers
the two **constant-rate** models: the birth–death process and its pure-birth special case, **Yule**.
Both run backward and forward. The heterogeneous-rate variants — where the rates vary through time,
across lineages, or at a carrying capacity — live on the
[advanced diversification](advanced-diversification.md) page and are forward only.

| Model | Rates | Reach for it when |
| --- | --- | --- |
| **Birth–death** | constant λ, μ | the default way to make a dated tree |
| **Yule** | pure birth (μ = 0) | a tree with no extinction |

## The models

### Birth–death

Constant `birth` (λ) and `death` (μ). The default and the usual choice for a tree to simulate
genomes, traits, or sequences along. Backward it is a reconstructed tree conditioned on the number
of extant tips; forward it grows in time and keeps extinct lineages.

### Yule

`Yule(birth)` is the pure-birth special case of the birth–death process — extinction is switched off
(`death 0`). Use it when you want a clean tree with no extinct lineages. Like birth–death it runs
both backward and forward.

## Command line

Constant-rate birth–death is the default `--diversification` process, and the only one that also runs
backward. Set `--death 0` for Yule.

```bash
# constant-rate birth–death, backward (reconstructed, conditioned on 50 tips)
zombi2 species --birth 1 --death 0.3 --tips 50 --age 5 --seed 1 -o run/

# Yule (pure birth, no extinction), backward
zombi2 species --birth 1 --death 0 --tips 20 --age 5 --seed 1 -o yule/

# Yule, forward in time (grown to a fixed age)
zombi2 species --mode forward --birth 1 --death 0 --age 3 --seed 1 -o yule_fwd/
```

## Python

Models live in `zombi2.species` (and re-export at the top level, so `zombi2.BirthDeath` also works):

```python
from zombi2.species import BirthDeath, Yule, simulate_species_tree

# backward: a reconstructed tree conditioned on the number of extant tips
tree = simulate_species_tree(BirthDeath(1.0, 0.3), n_tips=20, age=5.0, seed=1)

# Yule is pure birth (no extinction); it runs backward or forward
yule = simulate_species_tree(Yule(1.0), n_tips=20, age=5.0, seed=1)
yule_fwd = simulate_species_tree(Yule(1.0), age=3.0, direction="forward", seed=1)
```

## Output

`species_tree.nwk` (the complete tree, extinct/ghost tips kept), and — when dead tips are present —
`species_tree_extant.nwk` pruned to the sampled-extant leaves. `species_nodes.tsv` gives per-node
metadata (`name`, `time`, `is_leaf`, `is_extant`), and `species_tree.log` is the run manifest.
Leaves use the [standard node names](../contributing/conventions.md#naming): `n*` extant, `e*`
extinct, `u*` unsampled, `i*` internal.

## Validation

- **Oracle.** The backward age sampler matches the reconstructed-process CDF by a KS test
  (`test_species_bd.py::test_sample_age_matches_cdf`); forward Yule growth matches `E[extant] =
  2·e^{λ·age}` (`test_species_forward.py::test_yule_has_no_extinction_and_matches_theory`).
- **Determinism.** A fixed `seed` reproduces the Newick exactly
  (`test_species_forward.py::test_reproducible`).
